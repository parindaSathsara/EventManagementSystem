01 - Executive Overview

Mobile + Web Proposal Vision

Customer Experience

Event discovery, smart filtering, seamless checkout, promo code application, booking history, and QR e-ticket access from a unified customer mobile app.

Supplier Workspace

Musicians, organizers, and managers can create events, configure ticket tiers, monitor sales, and request payouts with complete visibility.

Admin Command Center

Super Admin and Admin role controls for approvals, commission settings, refund handling, promotions, and operational analytics.

02 - Functional Scope

Core System Modules

Identity & Access

  • JWT authentication and role-based access
  • Password reset and account verification
  • Session controls and audit-ready access security

Event Lifecycle

  • Create, edit, review, approve, publish workflows
  • Event categories, schedules, media, and status control
  • Ticket tiering and optional seat allocation

Ticketing Engine

  • Unique ticket IDs and QR code generation
  • E-ticket delivery and validation state tracking
  • Cancellation and refund logic with traceability

Payments

  • Cards, bank transfer, wallets, and Koko Pay
  • Webhook-based payment confirmations
  • Refund pipeline, logs, and reconciliation support

Promotions

  • Fixed and percentage promo code models
  • Early bird offers and event-specific campaigns
  • Usage limits, expiry, and fraud-resistant rules

Analytics & Reports

  • Revenue, tickets sold, and conversion data
  • Supplier earnings and commission performance
  • Admin and supplier export-ready dashboards

Use Case Specification

System Actors & Use Cases

System Actors

👤

Customer

A user who browses events and purchases tickets using the mobile application.

🎵

Supplier

An event organizer or musician who creates and manages events via the supplier dashboard.

🛡️

Admin

A system controller who manages users, events, promotions, reports, and platform configurations.

⚡

Super Admin

An advanced administrative role with additional privileges including admin account management and role control.

03 - Architecture

Proposed Technical Stack

Frontend Layer

React JS for web dashboards and React Native for Android/iOS mobile apps with shared API integration.

API & Services

Laravel backend architecture with modular REST APIs, secure webhook handling, and service-level business logic.

Data & Infra

MySQL primary database, Redis caching, media object storage, CI/CD pipeline, and cloud deployment on AWS/Azure/GCP.

HTTPS Enforcement OWASP Best Practices Rate Limiting Audit Logs Disaster Recovery
